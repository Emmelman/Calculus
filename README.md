# 🦉 Умножариум

Красочный интерактивный тренажёр **таблицы умножения** для детей 7–9 лет.
Работает офлайн, масштабируется на телефон и планшет, ставится на Android как
приложение (PWA), а позже заворачивается в настоящий APK через Capacitor.

## Что внутри

- 🧠 **Учить** — наглядные модели: группы точек, счёт через шаг, числовая прямая.
- ❓ **Викторина** — выбор ответа из вариантов.
- ⚡ **Гонка** — на скорость за 60 секунд, с комбо.
- 🃏 **Найди пару** — мемори: пример ↔ ответ.
- 🏆 **Геймификация** — монеты, уровни, серии (streak), достижения, конфетти, звуки.
- 🎯 **Адаптивная сложность** — система Лейтнера: тяжёлые и недавно ошибочные примеры
  (например, 7×8) показываются чаще.
- 👨‍👩‍👧 **Экран для родителя** — карта освоения, статистика по дням, «над чем поработать»,
  защита PIN-кодом.
- 🤖 **ИИ-помощник (опционально)** — задачки-истории по теме и объяснение ошибок через
  OpenRouter. Если выключен или нет интернета — работают встроенные шаблоны.

## Быстрый старт (разработка)

```bash
npm install
npm run dev          # http://localhost:5173 (откроется и на телефоне/планшете в той же сети)
```

Проверки:

```bash
npm test             # unit-тесты доменной логики (факты, освоенность, планировщик)
npm run build        # типизация + production-сборка в dist/
npm run preview      # посмотреть собранную версию
```

## ИИ-помощник (необязательно)

Ключ OpenRouter держится **только на сервере-прокси** и никогда не попадает в приложение.

```bash
cp server/env.example server/.env      # впишите OPENROUTER_API_KEY
npm run dev:all                        # поднимет приложение + прокси (порт 8799) вместе
# или по отдельности:
npm run server                         # только прокси
```

Vite в режиме разработки проксирует `/api/*` на прокси. Без ключа приложение
полностью работает — ИИ-функции мягко используют локальные шаблоны.

> ⚠️ Репозиторий публичный. `server/.env` в `.gitignore` — **не коммитьте реальный ключ.**
> Подключение ChatGPT из `~/Oroboros` (личный Codex OAuth) для приложения **не** используется:
> тот токен привязан к личной подписке и небезопасен для распространения.

## Установка на Android как PWA

1. Соберите и поднимите: `npm run build && npm run preview` (или разместите `dist/` на хостинге).
2. На Android откройте адрес в Chrome → меню → **«Добавить на главный экран»**.
3. Иконка появится на рабочем столе, приложение работает офлайн.

## Сборка APK на планшете (Termux + Ubuntu)

На устройстве с Android SDK (например, планшет: Termux → Ubuntu proot, где много памяти):

```bash
# 1. Поставить инструменты в Ubuntu-proot: Node 20, JDK 17, Android SDK + platform-tools.
# 2. ИИ-помощник: прокси (server/) живёт на сервере за nginx (:443 →
#    /umnozharium/api/ → 127.0.0.1:8799). Наружу порт не торчит — планшет
#    дотягивается через SSH-туннель, поэтому APK обращается к 127.0.0.1:
ssh -N -L 8443:127.0.0.1:443 ПОЛЬЗОВАТЕЛЬ@СЕРВЕР &   # держать туннель при работе помощника
# 3. В проекте:
npm install
npm install @capacitor/core @capacitor/cli @capacitor/android
# URL прокси зашивается в бандл при сборке (без него помощник мягко уходит в
# офлайн-шаблоны, ядро-тренажёр работает всегда):
export VITE_HELPER_URL="https://127.0.0.1:8443/umnozharium"
npm run build
npx cap add android                 # создаст папку android/ (она в .gitignore)
bash scripts/android-apply-config.sh  # доверие самоподписанному серту прокси
npx cap sync
cd android && ./gradlew assembleDebug
# APK: android/app/build/outputs/apk/debug/app-debug.apk
```

Шаг `android-apply-config.sh` встраивает `android-config/network_security_config.xml`
и серт `helper_ca.crt` в проект, чтобы APK доверял TLS-серту nginx **только** для
`127.0.0.1`/`localhost`. Серт самоподписанный (CN=nikapp-dev, SAN включает
`127.0.0.1`); если серт на сервере перевыпустят — обновите
`android-config/helper_ca.crt` и пересоберите APK. При переезде на домен +
Let's Encrypt этот шаг не нужен (системные CA уже в доверии): достаточно сменить
`VITE_HELPER_URL` на `https://ДОМЕН/umnozharium`.

Либо просто запустите `npm run dev` и откройте приложение в браузере планшета —
это самый быстрый способ проверить адаптив на реальном устройстве.

## Архитектура

```
src/
  domain/     чистая логика (без React) + тесты: facts, mastery (Leitner), scheduler
  store/      Zustand + persist(localStorage): прогресс, настройки; навигация
  components/ переиспользуемые виджеты (клавиатура, маскот, кольцо прогресса …)
  screens/    Home, Learn, Quiz, Speed, Memory, Result, Parent, Settings
  helper/     клиент ИИ-помощника с офлайн-fallback
  lib/        звук (Web Audio), конфетти, тосты-награды
server/       прокси OpenRouter (держит ключ), /api/helper/*  (необязателен)
```

Принципы: офлайн-first, ключ ИИ только на прокси (не в приложении), адаптивная
вёрстка через `clamp()` и CSS Grid (телефон + планшет), детерминированная и
покрытая тестами доменная логика.

## Стек

React 18 · TypeScript · Vite · Zustand · vite-plugin-pwa · Express (прокси) ·
canvas-confetti · Vitest · Capacitor (для APK).
